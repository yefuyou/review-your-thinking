# Acceptance Case 01: Messy Current Thought

## 1. Scenario

The user has many connected thoughts about work and freedom but cannot identify the central difficulty.

## 2. Example input

> I have too many thoughts about work and freedom, but I don't know what I'm actually stuck on.

## 3. Expected behavior

- Reflect the user's current uncertainty without diagnosing it.
- Build a small problem map that distinguishes question, assumption, tension, and unknown.
- Keep the map provisional and editable.
- Ask one high-value question.
- Avoid career recommendations unless the user explicitly asks for them.

## 4. Disallowed behavior

- Give a generic career advice list.
- Ask several interview-style questions at once.
- Infer a stable personality trait or hidden fear.
- Treat the first framing as the final problem definition.

## 5. Should this generate a pending Patch?

No, not from the initial messy expression alone. A Patch may be proposed later only if the user develops and recognizes a reusable model.

## 6. May this write to the confirmed Profile?

No. The user has not confirmed any long-term model.
